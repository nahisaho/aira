# AIRA 憲法 (9 か条)

MUSUBIX SDD 方法論に基づくプロジェクト憲法。

## 第一条: ライブラリファースト
コア機能はライブラリとして設計し、UI 層から分離する。

## 第二条: CLI インターフェース
主要操作は CLI からも実行可能にする。

## 第三条: テストファースト (TDD)
実装前にテストを記述する (Red-Green-Blue サイクル)。

## 第四条: EARS 形式要件
すべての要件は EARS 形式で記述する。

## 第五条: トレーサビリティ
REQ → DES → TSK → Code → Test の 100% トレーサビリティを保証する。

## 第六条: プロジェクトメモリ
steering/ ディレクトリを常に参照・更新する。

## 第七条: 設計パターン
C4 モデルで設計を文書化し、既知のパターンを優先する。

## 第八条: 意思決定記録
重要な技術的決定は ADR として docs/adr/ に記録する。

## 第九条: 品質ゲート
lint・型チェック・テストがすべて pass してからマージする。
